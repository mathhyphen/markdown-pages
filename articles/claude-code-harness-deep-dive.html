Claude Code 架构深度解析

基于 SinghCoder/claude-code 源码 | 51万行代码核心设计

Claude Code 的核心竞争力不在于「模型有多强」，而在于「如何让模型在编程场景下持续、稳定、可控地发挥能力」。本文深入解析其 Agent Harness 的核心工程设计。

Claude Code 的核心是一个有状态的持久化 Agentic Loop，而非一次性的代码生成器。

Compaction 四层策略

非破坏性

  • Microcompact：折叠重复工具输出
  • Autocompact：摘要历史

破坏性

  • History Snip：截断最旧消息
  • Reactive：413 错误后触发
核心洞察：Claude Code 的 Compaction 不是等 context 满了才救火，而是有多级预警机制主动管理。

Mailbox 通信机制

Agent 之间不直接通信，而是通过 mailbox 文件异步协作：

SendMessageTool → teammate_mailbox.js → 文件持久化
                                          ↓
对方轮询 → 读取文件 + teammate_mailbox attachment → 通知
优势：Agent 可以完全异步工作，不需要实时连接。父 Agent 终止时通过 AbortController 信号传播到所有子 Agent。

Yoga 布局引擎

Claude Code 使用纯 TypeScript 实现的 Meta Yoga 布局引擎：

  • CSS Flexbox 子集：flex-direction, flex-grow, align-items, justify-content
  • Measure Functions：文本节点动态高度计算
  • 布局缓存优化：_hasL 单槽缓存
为什么手感好：不是重新渲染整个屏幕，只计算「哪些部分变了」。diff 思维让每次按键都精准刷新，无闪烁。

MEMORY.md 索引限制

  • 最多 200 行 或 25KB
  • 超限时追加 > WARNING 提示 Agent 精简
重要洞察：RAG 不是堆砌所有信息，而是「精准提取 + 结构化索引 + 严格限制」。

输出持久化策略

1. 大小估计 → getContentSizeEstimate()
2. 超阈值 → truncateMcpContentIfNeeded()
3. 二进制/大文本 → persistBinaryContent() / persistToolResult()
4. 模型收到指针 + 访问说明，而非完整内容

十大设计理念总结

理念核心价值
Query Guard状态机防止并发请求混乱
Streaming First延迟感知比实际延迟更重要
Permission Gating安全要从第一天设计
Token Budget资源限制要内置而非外挂
Compaction主动管理，多级预警
Memory Taxonomy结构化记忆 > 堆砌信息
Hook Lifecycle扩展不靠修改核心代码
Sidechain隔离关注点，防止 context 膨胀
Mailbox异步解耦，分布式通信
Damage Trackingdiff 思维，极致性能
工程哲学：Claude Code 的成功不在于使用了多强的模型，而在于对「如何让模型在编程场景下持续、稳定、可控地发挥能力」这件事的深刻理解和精密工程实现。
