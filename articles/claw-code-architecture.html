从泄露源码到完整复刻：一个项目如何深度解构 Claude Code

Claude Code 源码泄露引发关注后，社区没有停留在围观层面。本文深度解析 instructkr/claw-code 项目如何从零复刻这个 Agent Harness，从架构设计到工程细节，揭示其完整的设计逻辑。

1. 项目概述

Claw Code 是对原始 TypeScript 版 Claude Code Agent Harness 的深度复写实现。当 Claude Code 源码疑似泄露后，instructkr 团队选择了一条独特的路径——不是直接 fork 或复制，而是重新理解、重新设计这个 Agent Harness 的核心逻辑。

背景：2025年底，Claude Code 源码疑似泄露引发了社区广泛讨论。instructkr/claw-code 项目正是在这个背景下诞生，目标是深度解析 Agent Harness 的工程设计，而非简单的代码复制。项目当前正在向高性能 Rust 运行时过渡（位于 dev/rust 分支）。

1.1 开发策略：OmX 编排

重写过程采用 oh-my-codex (OmX) 工作流驱动：

  • $team 模式：并行评审，允许多个执行单元同时工作
  • $ralph 模式：持久执行循环，配合架构级验证实现渐进式开发

2. 核心架构

Claw Code 采用分层架构，从上到下分为：自然语言空间、执行运行时层、命令工具层和引导初始化层。

1 CLI 入口与数据模型

最顶层是用户交互入口点，包含主入口 (src/main.py) 和核心数据结构 (src/models.py)。

2 Port 与 Context 管理

Port Manifest (src/port_manifest.py) 和 Context (src/context.py) 负责管理端口抽象和运行时上下文。

3 运行时与执行引擎

PortRuntime (src/runtime.py) 和 QueryEnginePort (src/query_engine.py) 构成核心执行引擎。

4 命令与工具注册表

Command Registry (src/commands.py)、Tool Registry (src/tools.py) 和 Permission System (src/permissions.py)。

5 引导与初始化

Setup (src/setup.py)、Prefetch (src/prefetch.py) 和 Bootstrap Graph (src/bootstrap_graph.py)。

3. 关键组件详解

3.1 PortRuntime：高层编排层

PortRuntime 是 Claw Code 环境的中央编排层，负责管理执行循环、路由用户查询到适当的处理程序，并协调工具执行。

3.2 QueryEnginePort：会话生命周期

QueryEnginePort 负责维护对话历史、管理消息历史，并通过 from_saved_session() 方法支持会话恢复。

3.3 ToolRegistry：工具快照系统

Claw Code 维护一个 tools_snapshot.json 文件，包含超过 100 个工具的元数据。工具发现函数支持：

  • get_tool(name)：大小写不敏感的工具查找
  • find_tools(query, limit)：模糊搜索
  • tool_names()：列出所有注册工具

3.4 组件关系表

组件 文件 职责
PortRuntime src/runtime.py 中央编排器，管理执行循环和查询路由
QueryEnginePort src/query_engine.py 对话历史、会话生命周期、消息管理
ToolRegistry src/tools.py 工具发现、快照加载、元数据管理

TranscriptStore src/transcript.py 对话记录追加和刷新

5.1 七阶段引导图

启动过程被概念化为一个七阶段管道，由 BootstrapGraph 管理，确保依赖关系被正确遵守。

6. Parity Audit 系统

这是 Claw Code 区别于普通重写的关键特性——内置的等效性审计工具。

Parity Audit 目的：测量 Python 工作空间与原始 TypeScript 快照在文件表面、命令和工具方面的匹配程度。

6.1 审计范围

  • 文件表面审计：检查文件结构是否与原版一致
  • 命令审计：验证命令定义完整性
  • 工具审计：确认 100+ 工具的映射状态

6.2 引用数据快照

src/reference_data/ 目录包含：

  • tools_snapshot.json：100+ 工具元数据
  • commands_snapshot.json：命令定义快照
  • files_surface.json：文件表面结构

7. 优缺点分析

优势

  • 深度复刻设计：避免直接复制，在理解原版基础上重新实现
  • 架构一致性：通过 Parity Audit 确保与原版功能对等
  • 多语言支持：Python 实现更易于 Python 生态集成
  • 持久会话：支持跨 CLI 调用的状态恢复
  • 权限系统：细粒度的工具访问控制
  • Rust 迁移路径：dev/rust 分支提供性能升级路径

劣势

  • 性能限制：Python 实现相比 TypeScript 原版可能有性能差距
  • 维护成本：需要同步原版 Claude Code 的更新
  • 功能覆盖：重写过程中可能遗漏原版某些边界情况
  • 社区规模：相比官方实现，社区支持和文档较少
  • 复杂度：Parity Audit 系统增加了额外维护负担

8. 总结

Claw Code 代表了一种独特的 Agent Harness 工程方法——当 Claude Code 源码泄露引发关注后，社区没有停留在围观层面，而是从零深度理解其架构设计。

其核心价值在于：

  • 揭示设计逻辑：提供了 Claude Code 内部工作原理的详细文档
  • 工程严谨性：通过 Parity Audit 系统确保与原版的功能等效性
  • 生态贡献：为 Python 生态提供原生集成能力，降低参与门槛
  • 性能演进：通过 Rust 迁移路径提供性能升级方向
启示：源码泄露固然引发关注，但真正有价值的是像 Claw Code 这样，用工程方法深入理解、设计和演进这个系统。
