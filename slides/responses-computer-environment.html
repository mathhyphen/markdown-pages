核心变化：从“会答题”到“会工作”

Responses API 被配上了 shell、文件系统、数据库与受控网络，模型开始拥有持续工作的环境。

真正的 agent 需要工作区，而不只是工具列表。
shell、文件系统和数据库是 agent 的外部记忆系统。

为什么 shell 重要

真实任务不仅是算数，还包括构建、测试、抓日志、转格式、管理文件和运行脚本。

真正的 agent 需要工作区，而不只是工具列表。
shell、文件系统和数据库是 agent 的外部记忆系统。
第 3 页

状态应该留在环境里

长输出落盘、中间结果留文件，模型只在需要时读取，这比把一切塞进上下文稳得多。

shell、文件系统和数据库是 agent 的外部记忆系统。
权限与策略设计是 agent 平台的核心组成，不是附加安全条款。
第 4 页

长任务靠 compaction，不靠上下文无限膨胀

系统会持续重写工作记忆，让 agent 带着更干净的状态继续前进。

权限与策略设计是 agent 平台的核心组成，不是附加安全条款。
compaction 不是锦上添花，而是长任务生存条件。
