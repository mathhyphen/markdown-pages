agent 不是普通 API

它需要流式事件、审批暂停、diff、恢复连接和长期线程，这些都超出了简单 request/response。

第 3 页

三层语义很关键

item 是原子事件，turn 是一次任务，thread 是持久会话；这让 UI 与状态恢复都变得稳定。

持久线程是长会话 agent 的基础设施，不是附加功能。
越 rich 的客户端体验，越需要清晰的协议原语。
第 4 页

为什么不用更轻的协议

MCP 或轻量 SDK 适合单点集成；完整产品形态则需要 richer session semantics。

越 rich 的客户端体验，越需要清晰的协议原语。
Web 客户端应尽量无状态，任务状态留在服务端。
第 5 页

我的结论

App Server 本质上是在把“agent 产品协议”这件事工程化、标准化。

Web 客户端应尽量无状态，任务状态留在服务端。
如果你在做真正的 agent 产品，协议设计与模型能力同样关键。
